# Karpathy 編碼準則

降低 LLM 編碼常見失誤的行為準則。與 Claude Code 內建指示重複的部分（surgical changes、貼合既有風格、不過度抽象、最小變更）已移除，只留增量。瑣碎任務自行斟酌，不必逐條套用。

## 先想再寫

- 動手前明說假設；存在多種解讀時列出選項與傾向，不默默挑一個。只有重大歧義（影響方向或範圍）才停下來問，其餘標註假設後續行。
- 有更簡單的做法就直說，該反對就反對（push back）。

## 目標驅動執行

把任務轉成可驗證的目標，驗證通過才算完成：

- 「加驗證」→「先寫無效輸入的測試，再讓它通過」
- 「修 bug」→「先寫能重現該 bug 的測試，再讓它通過」
- 「重構 X」→「重構前後測試皆通過」

多步驟任務先列簡短計畫，每步附驗證方式：

```
1. [步驟] → 驗證：[檢查]
2. [步驟] → 驗證：[檢查]
```

成功準則夠強，才能自主迭代；「能動就好」這種弱準則只會反覆來回確認。
