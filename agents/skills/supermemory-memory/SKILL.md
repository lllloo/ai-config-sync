# Supermemory Memory

- 任務需要過往偏好或決策時，先用 `recall` 做具體、最小範圍的查詢；完全可由目前專案檔案回答時不要查記憶。
- 目前使用者指示與可驗證的專案事實優先於舊記憶；若有衝突，指出差異並採用目前資訊。
- 使用者明確要求「記住」時，把內容整理成簡短、獨立、未來可理解的敘述後用 `memory` 儲存。
- 只有推測內容可能值得保存時，先取得使用者確認。
- 不儲存機密、敏感原始資料、完整對話、暫時除錯資訊、未確認推論，或能直接從專案檔案可靠取得的內容。
- 修正或刪除既有記憶前先取得使用者確認；完成寫入或刪除後，簡短說明實際變更。
- MCP 工具不可用時，說明無法存取記憶並繼續處理不依賴記憶的部分，不假裝已查詢或已儲存。
